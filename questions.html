Part A: True / False (50 Questions)

1

Software engineering only deals with writing code.

Answer: FALSE
✔ Correct Statement: Software engineering deals with ALL aspects of software production — from early specification through to maintenance — not just coding.
2

System software runs independently without depending on other software.

Answer: TRUE
3

Application software depends on system software to run.

Answer: TRUE
4

Android is an example of application software.

Answer: FALSE
✔ Correct Statement: Android is an example of SYSTEM software — it provides a platform for other apps to run.
5

MATLAB is an example of engineering/scientific software.

Answer: TRUE
6

Embedded software cannot reside within hardware products.

Answer: FALSE
✔ Correct Statement: Embedded software DOES reside within hardware products (e.g., microwave oven controller) to implement and control their features.
7

Web applications are accessed using a web browser.

Answer: TRUE
8

MS Office is an example of system software.

Answer: FALSE
✔ Correct Statement: MS Office is an example of APPLICATION software — it is a stand-alone program that solves a specific business need.
9

Software engineering is a sub-field of computer engineering.

Answer: TRUE
10

Computer engineering deals only with software.

Answer: FALSE
✔ Correct Statement: Computer engineering deals with BOTH software AND hardware, including networking, processors, and databases.
11

Maintainability means the software can evolve to meet changing needs.

Answer: TRUE
12

Efficiency refers to how well software uses system resources.

Answer: TRUE
13

Acceptability means the software is acceptable only to developers.

Answer: FALSE
✔ Correct Statement: Acceptability means the software must be acceptable to the TYPE OF USERS for which it is designed — not just developers.
14

SDLC stands for Software Development Life Cycle.

Answer: TRUE
15

The first phase of SDLC is testing.

Answer: FALSE
✔ Correct Statement: The first phase of SDLC is PLANNING & REQUIREMENT ANALYSIS — the most important and necessary stage.
16

The SRS document is created during the Defining Requirements phase.

Answer: TRUE
17

Deployment happens before testing in the SDLC.

Answer: FALSE
✔ Correct Statement: TESTING happens before deployment. Only after testing confirms no bugs is the software deployed.
18

Maintenance is the shortest phase of SDLC.

Answer: FALSE
✔ Correct Statement: Maintenance is the LONGEST phase of SDLC — it covers the entire operational life of the software.
19

David Hooker proposed seven principles of software engineering.

Answer: TRUE
20

The KISS principle means 'Keep It Simple, Stupid!'

Answer: TRUE
21

Planning ahead for reuse increases the cost of software components.

Answer: FALSE
✔ Correct Statement: Planning ahead for reuse REDUCES the cost and increases the value of both the reusable components and the systems that use them.
22

A software myth is a correct belief about software development.

Answer: FALSE
✔ Correct Statement: A software myth is a WRONG (incorrect) belief about software and the process used to build it.
23

Brook's Law states that adding programmers to a late project speeds it up.

Answer: FALSE
✔ Correct Statement: Brook's Law states: 'Adding manpower to a late software project makes it LATER' — new people need onboarding time.
24

Ambiguous requirements can lead to project disaster.

Answer: TRUE
25

Software changes introduced early in development are more expensive than late ones.

Answer: FALSE
✔ Correct Statement: Changes introduced EARLY (before design/code) are CHEAPER. Late changes (after coding) can cost 50-100x more.
26

Maintenance accounts for 60-80% of total software lifetime cost.

Answer: TRUE
27

Technical reviews can detect defects before code is executed.

Answer: TRUE
28

The only deliverable in a software project is the working program.

Answer: FALSE
✔ Correct Statement: A working program is only ONE part. Other deliverables include plans, documents, models, and test cases.
29

Heterogeneity in software means integrating systems across different platforms.

Answer: TRUE
30

A legacy system is a modern, up-to-date computing system.

Answer: FALSE
✔ Correct Statement: A legacy system is an OUTDATED computing system (hardware or software) that is still in use.
31

Security and trust is one of the three general issues in software engineering.

Answer: TRUE
32

Software engineers must respect client confidentiality even without a signed agreement.

Answer: TRUE
33

A software engineer should accept any work regardless of their competence.

Answer: FALSE
✔ Correct Statement: A software engineer should NOT knowingly accept work that is OUTSIDE their competence.
34

Intellectual property rights are given to persons over creations of their minds.

Answer: TRUE
35

Playing games on your employer's computer is not a form of computer misuse.

Answer: FALSE
✔ Correct Statement: Playing games on an employer's machine IS a form of computer misuse — even if trivial, it is still misuse.
36

The Waterfall Model was the first SDLC model developed.

Answer: TRUE
37

In the Waterfall Model, the next phase starts before the previous one is complete.

Answer: FALSE
✔ Correct Statement: In the Waterfall Model, the next phase starts ONLY AFTER the previous phase is completed and signed off.
38

The maintenance phase is the longest phase of the Waterfall Model.

Answer: TRUE
39

The Waterfall Model works well for complex, large projects.

Answer: FALSE
✔ Correct Statement: The Waterfall Model is NOT suitable for complex and large projects — it works best for small projects with fixed requirements.
40

The Iterative Waterfall Model provides feedback paths between phases.

Answer: TRUE
41

There is a feedback path from any phase back to the Feasibility Study in Iterative Waterfall.

Answer: FALSE
✔ Correct Statement: There is NO feedback path back to the Feasibility Study — once a project is accepted, it is not easily abandoned.
42

The Waterfall Model produces working software early in the lifecycle.

Answer: FALSE
✔ Correct Statement: The Waterfall Model produces NO working software until LATE in the lifecycle — only after all phases are complete.
43

The V-Shaped Model is an extension of the Waterfall Model.

Answer: TRUE
44

Verification asks 'Are we building the right product?'

Answer: FALSE
✔ Correct Statement: VALIDATION asks 'Are we building the right product?' — Verification asks 'Are we building the product RIGHT?' (per specifications).
45

Validation is done by executing code dynamically.

Answer: TRUE
46

In the V-Model, testing is associated with every phase of development.
