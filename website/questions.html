Part C: Short Questions (50 Questions)

1

Define Software Engineering.

Software Engineering is an engineering discipline concerned with all aspects of software production — from early-stage system specification through to maintaining the system after it has gone into use.
2

What is the difference between software and engineering?

Software is a collection of programs that accomplish tasks. Engineering is the application of scientific and practical knowledge to design and build solutions. Software Engineering combines both.
3

List and briefly describe the seven categories of software application domains.

1) System Software (OS), 2) Application Software (specific user tasks), 3) Engineering/Scientific Software (MATLAB), 4) Embedded Software (device controllers), 5) Web Applications (browser-accessed), 6) Business Software (ERP, banking), 7) AI Software (robotics, neural networks).
4

What is embedded software? Give one example.

Embedded software resides within a product or system and controls its features and functions. Example: The control software inside a microwave oven's keypad.
5

Differentiate between System Software and Application Software.

System software provides a platform for other software and manages hardware resources (e.g., Linux). Application software is built for specific user tasks and depends on system software (e.g., Photoshop).
6

What are the four essential attributes of good software?

1) Maintainability — software can evolve with changing needs. 2) Dependability and Security — reliable, safe, and secure. 3) Efficiency — doesn't waste memory/processing. 4) Acceptability — usable and understandable by target users.
7

What is Maintainability in software?

Maintainability means the software is written in such a way that it can evolve to meet the changing needs of customers without excessive effort or risk.
8

Define SDLC and explain why it is needed.

SDLC (Software Development Life Cycle) is a structured process for developing software systematically. It is needed to ensure all team members follow the same process, prevent chaos, and deliver quality software on time.
9

List all 7 phases of SDLC.

1) Planning & Requirement Analysis, 2) Defining Requirements (SRS), 3) Designing the Software, 4) Developing the Software, 5) Testing, 6) Deployment, 7) Maintenance.
10

What is an SRS document?

An SRS (Software Requirement Specification) document contains all product requirements to be constructed and developed during the project lifecycle. It serves as a contract between client and developer.
11

What is a Feasibility Study?

A Feasibility Study is an assessment of the practicality of a proposed project. It helps decision-makers determine whether the project is likely to be successful before resources are committed.
12

State the 7 principles of Software Engineering by David Hooker.

1) The Reason It All Exists, 2) KISS, 3) Maintain the Vision, 4) What You Produce Others Will Consume, 5) Be Open to the Future, 6) Plan Ahead for Reuse, 7) Think!
13

Explain the KISS principle.

KISS stands for 'Keep It Simple, Stupid!' All design should be as simple as possible. Simpler systems are easier to understand, test, maintain, and extend over time.
14

What does 'Plan Ahead for Reuse' mean?

Build software so that code and design components can be reused in other projects. This reduces cost and increases value, as the same component doesn't need to be rebuilt from scratch.
15

What is a Software Myth?

A software myth is a wrong belief about software and the process used to build it. Myths lead to poor management decisions and project failures.
16

Explain Brook's Law.

Brook's Law states: 'Adding manpower to a late software project makes it later.' New team members need time to onboard, and senior developers must train them instead of coding.
17

What is the impact of late requirement changes?

Requirements changes have greater cost impact when introduced late in development. Early changes (before design/code) cost little; changes after coding/testing can cost 50-100x more.
18

What is a Technical Review?

A technical review is a quality assurance mechanism applied from the inception of a project to detect defects early through inspection of code, design, or requirements — without executing the code.
19

What are the three General Issues in Software Engineering?

1) Heterogeneity — operating across different platforms and legacy systems. 2) Business & Social Change — need to rapidly evolve software. 3) Security & Trust — protecting software from malicious attacks.
20

What is a Legacy System?

A legacy system is any outdated computing system (hardware or software) that is still in use despite being technologically obsolete. It poses integration challenges for new software.
21

List the four ethical responsibilities of a Software Engineer.

1) Confidentiality — respect client/employer secrets. 2) Competence — don't misrepresent skills. 3) Intellectual Property Rights — respect legal protections. 4) Computer Misuse — don't misuse technical access.
22

What is Computer Misuse in software ethics?

Computer misuse means using technical skills to harm or access other people's computers without authorization. It ranges from trivial acts (games on employer's machine) to serious crimes (spreading malware).
23

Define Intellectual Property Rights (IPR).

IPR are the rights given to persons over the creations of their minds. In software, this includes patents and copyright over code, algorithms, and designs.
24

What is the Waterfall Model?

The Waterfall Model is the first SDLC model, following a strictly sequential process where the output of one phase becomes the input for the next, and no phase begins until the previous one is completed.
25

List the phases of the Waterfall Model.

1) Feasibility Study, 2) Requirements Analysis & Specification, 3) System & Software Design, 4) Coding & Unit Testing, 5) Integration & System Testing, 6) Maintenance & Operation.
26

When should the Waterfall Model NOT be used?

Waterfall should not be used for complex projects, projects with ambiguous or frequently changing requirements, or projects where early working software is needed.
27

State two advantages and two disadvantages of the Waterfall Model.

Advantages: Simple to use; release date and cost can be determined before development. Disadvantages: Cannot accommodate requirement changes; no working software produced until late.
28

What is the Iterative Waterfall Model?

The Iterative Waterfall Model is an improved version of the Classic Waterfall Model that provides feedback paths from every phase back to its preceding phase, allowing error correction.
29

Why is there no feedback path to the Feasibility Study in Iterative Waterfall?

Because once a project has been accepted and started, organizations do not easily abandon it. Therefore, there is no mechanism to return to the feasibility stage.
30

What is the major disadvantage of Iterative Waterfall?

All requirements must be clear before development starts. After development begins, no changes can be incorporated. Also, there is no incremental delivery — customers must wait for the full product.
31

What is the V-Shaped Model?

The V-Shaped Model (Verification & Validation Model) is an extension of Waterfall where each development phase has a corresponding testing phase planned in parallel, forming a V shape.
32

Differentiate between Verification and Validation.

Verification: Checking if software is built as per specifications (static, no code execution). Validation: Checking if software meets client's actual needs (dynamic, code is executed).
33

What are the four development phases on the left side of the V-Model?

1) Requirements Analysis, 2) System Design, 3) Architecture Design, 4) Module Design. (Bottom: Implementation)
34

What are the four testing phases on the right side of the V-Model?

1) Unit Testing, 2) Integration Testing, 3) System Testing, 4) Acceptance Testing.
35

What testing corresponds to Architecture Design in the V-Model?

Integration Testing. The integration test plan is designed during the architecture design phase.
36

What testing corresponds to Requirements Analysis in the V-Model?

Acceptance Testing. The acceptance test plan is designed during requirements analysis.
37

State two advantages of the V-Shaped Model over the Waterfall Model.

1) Testing is planned in parallel with every development phase — defects found earlier. 2) Higher chance of success since testing planning happens well before coding.
38

What is the main disadvantage of the V-Shaped Model?

It is very rigid and least flexible. If requirements change mid-development, all test documents and requirement documents must also be updated, which is costly.
39

Describe the Incremental Model in simple terms.

The Incremental Model divides the software into standalone modules and delivers them one at a time. Each delivered increment adds new functionality, and customers can use the product early.
40

What is typically the 'first increment' in the Incremental Model?

The first increment is usually a core product containing the most basic and essential functionality that the customer can use while additional features are being built.
41

State three advantages of the Incremental Model.

1) Errors are easy to recognize. 2) Tolerates changing requirements. 3) Client gets important functionality early without waiting for the complete product.
42

State three disadvantages of the Incremental Model.

1) Needs good planning to divide work into increments. 2) Actual cost may exceed estimated cost. 3) A problem in one unit needs correction across all related units.
43

When is the Incremental Model most appropriate?

When the project has a lengthy development schedule, when quick delivery is demanded by the customer, when the team is less skilled, or when high-priority requirements need to be delivered first.
44

Compare customer interaction in Waterfall vs Incremental Model.

In Waterfall, customer interaction is only at the start (requirements) and end (delivery). In Incremental Model, customers interact throughout — providing feedback after every increment.
45

What is the full form of SDLC?

Software Development Life Cycle.
46

What is the role of a Business Analyst in the Planning phase?

The Business Analyst meets with the client to gather requirements — understanding what to build, who the end users are, and what the product's objective is.
47

What does 'Be Open to the Future' mean as a software engineering principle?

Systems must be designed to easily adapt to changes — specifications may change at any time and hardware platforms can become obsolete quickly. Flexible design future-proofs the software.
48

Why is Maintenance the longest phase of the Waterfall Model?

Because after deployment, real-world usage reveals new bugs, new user requirements emerge over time, and the system needs constant improvements and updates — this ongoing process lasts the entire life of the software.
49

What is the difference between Unit Testing and Integration Testing?

Unit Testing verifies each individual module works correctly in isolation. Integration Testing verifies that all the individual modules work correctly together when combined.
50

What is System Testing?

System Testing is testing of the complete, integrated software system to verify it meets its specified requirements. It is done just before product release.
51

Why is it important to 'Think!' before coding?

Thinking through a problem before coding leads to better design, fewer mistakes, and a deeper understanding of the task. Even if thinking leads to a mistake, it becomes a valuable learning experience.
