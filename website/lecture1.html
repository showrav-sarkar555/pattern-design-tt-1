Lecture 1: Introduction to Software Engineering

1. What is Software Engineering?

What is Software?

A collection of programs that accomplish any particular task. Software is not just code — it includes documentation, data, and the procedures that govern its use.

What is Engineering?

The application of scientific and practical knowledge to invent, design, build, maintain, and improve frameworks, processes, systems, and products.

Definition of Software Engineering

"Software engineering is an engineering discipline that is concerned with all aspects of software production from the early stages of system specification through to maintaining the system after it has gone into use."

💡 Real-World Example Building a banking app (like bKash or Nagad) requires gathering user requirements, designing the architecture, writing secure code, testing transactions, deploying to servers, and continuously maintaining it — all of these together form software engineering.

2. Software Application Domains (7 Categories)

1. System Software

Software designed to provide a platform for other software or to service other programs. It interacts directly with the hardware.

  • Examples: Windows, Linux, macOS, Android, iOS, device drivers
  • Written in: C, C++, Assembly

2. Application Software

Stand-alone programs that solve a specific business need for the end user.

  • Examples: MS Office, Google Chrome, Adobe Photoshop, Skype, Notepad
💡 Example When you use Microsoft Word to write a report, you are using application software. The underlying Windows OS is system software providing Word a platform to run.

3. Engineering / Scientific Software

Software satisfying the needs of scientific or engineering users to perform enterprise-specific calculations or simulations.

  • Examples: MATLAB, AutoCAD, PSPICE, ORCAD, ANSYS

4. Embedded Software

Software that resides within a product or system and is used to implement and control features and functions. Usually has strict real-time constraints.

  • Examples: Microwave oven keypad controller, washing machine controller, ABS braking system in cars
💡 Example The software inside your refrigerator that controls temperature and shows alerts on a screen is embedded software. It runs on a tiny microcontroller permanently fixed inside the appliance.

5. Web Applications

Software accessed using a web browser, providing features and content to end users, often integrated with corporate databases.

  • Examples: Online shopping carts, Google Docs, YouTube, online banking portals

6. Product-Line / Business Software

The most widely used category; supports business operations and enterprise workflows.

  • Examples: ERP systems, hospital management software, inventory management, accounting software

7. Artificial Intelligence Software

Computer programs that mimic human behavior by learning from data patterns and insights.

  • Examples: Robotics, expert systems, face recognition, voice assistants (Siri, Alexa), ChatGPT

5. Essential Attributes of Good Software

1. Maintainability

Software should be written so that it can evolve to meet changing needs of customers. This is critical because software change is an inevitable requirement of a changing business environment.

💡 Example When WhatsApp added the "Status" feature, developers had to modify existing code without breaking the chat functionality — this is maintainability in action.

2. Dependability and Security

Software dependability includes reliability, security, and safety. Dependable software should not cause physical or economic damage in the event of system failure. Malicious users should not be able to access or damage the system.

💡 Example An ATM software must be dependable — if it crashes during a transaction, it must not deduct money without dispensing cash. It must also be secure against hacking attempts.

3. Efficiency

Software should not make wasteful use of system resources (memory, processor cycles). Efficiency includes responsiveness, processing time, and memory utilization.

💡 Example A web page that loads in 1 second is more efficient than one that takes 10 seconds. Google's search engine is optimized to return results in milliseconds.

4. Acceptability

Software must be acceptable to the type of users for which it is designed. It must be understandable, usable, and compatible with other systems they use.

💡 Example A hospital management software must have a simple, clear UI that doctors and nurses (not programmers) can use without extensive training.
